함수의 정의

Javascript에서 함수는 function 키워드를 사용하여 정의됩니다.
함수는 입력(매개변수)을 받아 작업을 수행하고, 결과를 출력(반환값)할 수 있습니다. 

함수 선언 방식 예시:
        function gteet(name) {
            return "안녕하세요, " + name + "님!";
        }
        console.log(greet("뤼튼"));
        이 예시에서 greet 함수는 name이라는 매개변수를 받아 인사말을 반환합니다.

        

    함수 표현식 방식 예시:
        var add = function(a,b) {
            return a + b;
        };
        console.log(add(5, 3));

백틱(``)

자바스크립트에서 백틱(``)은 템플릿 리터럴을 사용하여 문자열을 표현하는 데 사용되는 특수 문자입니다.
백틱을 사용하면 문자열 내에서 변수나 표현식을 쉽게 삽입할 수 있고, 여러 줄에 걸친 문자열을 간편하게 작성할 수 있습니다.📝
문자열 내에서 변수의 값을 쉽게 삽입할 수 있으며, ${변수명} 형태로 사용합니다.
이 방법은 코드를 더욱 간결하고 읽기 쉽게 만들어 줍니다. 

{
        const name = "뤼튼";
        const greeting = `안녕하세요, ${name}입니다!`;
        console.log(greeting);
}
안녕하세요, 뤼튼입니다!

01. 함수 : 선언적 함수

선언적 함수는 이름을 가진 함수를 선언하는 방식입니다.
이 방식은 함수를 선언할 때 바로 이름을 부여하고, 이 이름을 사용하여 함수를 호출할 수 있습니다.
즉, 변수에 할당하지 않고도 함수를 정의하고 사용할 수 있는 것이죠! 😊
코드 읽는 순서와 상관없이 함수가 먼저 생성됩니다. 

{
        function func() {
            console.log("01. 함수가 실행되었습니다.");
        }
        func();
}

02. 함수 : 익명 함수

익명 함수는 이름이 없는 함수를 말합니다. 일반적으로 변수에 할당하여 사용되며, 이 변수를 통해 함수를 호출할 수 있습니다.
익명 함수는 주로 일회성 함수로 사용되거나, 콜백 함수로 활용되는 경우가 많아요.
변수에 할당하여 사용되며, 코드 실행 순서에 따라 함수가 생성됩니다. 

{
        const func = function () {
            console.log("02. 함수가 실행되었습니다.");
        };
        func();
}

03. 함수 : 매개변수 함수

매개변수 함수는 함수가 호출될 때 전달되는 값을 받아들이는 변수입니다.
함수를 정의할 때, 매개변수를 선언하여 함수 내에서 사용할 수 있습니다.
매개변수는 함수 내에서 지역 변수로 취급되며, 함수 외부에서는 접근할 수 없습니다. 

{
        // 선언적 함수
        function func(str) {
            console.log(str);

        func("03. 함수가 실행되었습니다.")

        // 익명 함수
        const func1 = function (str) {
            console.log(str);
        }
        func1("03. 함수가 실행되었습니다.");
}

04. 함수 : 리턴값 함수

함수의 리턴값은 함수가 호출될 때 생성된 출력 또는 결과를 말합니다.
이는 숫자, 문자열, 리스트 튜플, 사전 또는 사용자 정의 객체 등 다양한 데이터 유형이 될 수 있습니다.
함수에서 특정 값을 반환하고 싶을 때는 return 키워드를 사용하며, return 문이 실행되면 함수는 종료되고 리턴값이 함수를 호출한 곳으로 전달됩니다. 

{
        // 선언적 함수
        function func() {
            return "04. 함수가 실행되었습니다."
        }
        console.log(func());

        // 익명 함수
        const func1 = function () {
            return "04. 함수가 실행되었습니다.";
        }
        console.log(func1());
}

05. 함수 : 매개변수 + 리턴값 함수

매개변수와 리턴값을 가지는 함수는 프로그래밍에서 매우 중요한 역할을 합니다.
이러한 함수는 입력값을 받아들이고, 그 입력값을 기반으로 작업을 수행한 후 결과를 반환합니다.
매개변수(Parameter)는 함수가 호출될 때 전달되는 값을 받아들이는 변수입니다.
리턴값(Return Value)은 함수가 수행된 후 반환되는 값입니다. 함수 내에서 작업을 수행한 후, 그 결과를 리턴하여 함수를 호출한 곳으로 전달합니다. 

{
        // 선언적 함수
        function func(str) {
            return str;
        }
        console.log(func("05. 함수가 실행되었습니다."));

        // 익명함수
        const func1 = function (str) {
            return str;
        }
        console.log(func1("05. 함수가 실행되었습니다."));
}

06. 화살표 함수 : 선언적 함수

화살표 함수는 ES6에서 도입한 새로운 함수 표현 방식으로, => 기호를 사용하여 정의합니다.
화살표 함수는 간결한 문법을 제공하며, 주로 콜백 함수나 간단한 함수 표현에 사용됩니다.
자신만의 this를 생성하지 않고, 함수가 정의된 스코프의 this값을 그대로 사용합니다. 

{
        // 선언적 함수
        function func() {
            console.log("06. 함수가 실행되었습니다.");
        }
        func();

        // 선언적 함수 -> 화살표 함수
        func1 = () => {
            console.log("06. 함수가 실행되었습니다.");
        }
        func();
}

10. 화살표 함수 : 매개변수 + 리턴값

화살표 함수는 일반적인 함수와 마찬가지로 매개변수와 반환 값을 가질 수 있습니다.
매개변수는 함수를 호출할 때 전달되는 값으로, 함수 내부에서 사용됩니다.

        }
        console.log(func("10. 함수가 실행되었습니다."));

        // 리턴 생략
        const func3 = str => str;
        console.log(func3("10. 함수가 실행되었습니다."));

        // 선언적 함수(가독성X)
        func4 = str => str;
        console.log(func4("10. 함수가 실행되었습니다."));
}

18. 함수 유형 : 프로토타입 함수

Javascript에서 프로토타입(prototype)은 객체지향 프로그래밍의 상속을 구현하는 핵심 메커니즘입니다.
프로토타입 : javascript는 클래스 기반 언어가 아닌 프로토타입 기반 언어입니다.
이는 객체의 원형(prototype)을 이용하여 새로운 객체를 생성하는 방식을 의미합니다.
프로토타입을 통해 객체는 메소드와 속성을 상속받을 수 있습니다.
상속 메커니즘: JavaScript에서 모든 객체는 다른 객체의 프로토타입을 상속받을 수 있습니다.
이를 통해 부모 객체의 프로퍼티와 메소드를 자식 객체가 사용할 수 있게 됩니다.
이러한 상속 구조는 코드의 재사용성을 높이고, 객체 간의 관계를 효율적으로 구성할 수 있게 합니다. 

{
        // 함수 정의
        function Func(num, str) {
            this.num = num;
            this.str = str;
        }

        // 메서드 추가
        Func.prototype.result = function () {
            console.log(`${this.num}. ${this.str}`);
        }

        // 인스턴스 생성
        const info1 = new Func(21, "함수가 실행되었습니다.");
        const info2 = new Func(22, "함수가 실행되었습니다.");

        // 호출
        info1.result();
        info2.result();
}

이 방식은 객체를 생성하고 초기화하는 가장 간단한 방법 중 하나입니다.
메서드 추가: 객체 리터럴 내에서 함수를 프로퍼티 값으로 사용하여 메서드를 추가할 수 있습니다.

20. 함수 : 즉시실행 함수(IIFE, Immediately Invoked Function Expression)

즉시 실행 함수는 정의되자마자 바로 실행되는 자바스크립트 함수입니다.
이 함수는 이름이 있거나 익명일 수 있으며, 한 번만 호출될 수 있습니다.
IIFE는 코드를 함수 내부에 캡슐화함으로써 변수나 함수 이름 충돌을 방지하는 데 도움을 줍니다.

✅ 즉시 실행 함수의 특징
🎈정의와 동시에 실행 : 즉시 실행 함수는 정의되는 즉시 실행됩니다.
🎈한 번의 실행 : 이 함수는 한 번의 실행 이후 사라지기 때문에 단 한 번의 사용이 필요한 경우에 적합합니다.

✅ 즉시 실행 함수의 장점
🎈전역 변수 생성 감소 : 전역 변수의 생성을 줄여 코드의 충돌 가능성을 낮춥니다.
🎈Private 변수 생성 : private한 변수를 만들어 외부에서의 접근을 제한할 수 있습니다.

* 전역 변수란 ?
프로그램의 어느 곳에서나 접근할 수 있는 변수, 이는 함수 밖에서 선언되며 프로그램 전체에서 사용할 수 있습니다.
반면, 지역변수는 특정 함수 내에서만 사용되고 그 함수 밖에서는 접근할 수 없습니다.
전역 변수는 프로그램이 종료될 때까지 존재하지만, 지역 변수는 함수 호출이 끝나면 사라집니다.

즉시 실행 함수는 코드의 안정성을 높이고, 전역 네임스페이스를 깨끗하게 유지하는데 도움을 줄 수 있습니다. 

{
        (function () {
            console.log("15. 실행되었습니다.");
        })();

        (() => {
            console.log("16. 실행되었습니다.");
        })();
}

21. 함수 : 파라미터 함수

파라미터 함수는 프로그래밍에서 매우 중요한 개념 중 하나로, 함수에 입력으로 제공되는 데이터를 가리키는 변수입니다.
파라미터는 함수의 기능을 확장하고, 프로그램의 유연성을 높이는 데 중요한 역할을 합니다.
파라미터(매개변수) : 함수 선언 시, 괄호 안에 정의된 변수로, 함수 내에서 입력값으로 사용됩니다.
인자(전달인자) : 함수를 호출할 때, 실제로 함수에 전달되는 값입니다. 파라미터를 통해 함수 내부로 전달됩니다.

✅파라미터 역할과 중요성
🎈데이터 전달: 파라미터 함수는 함수 외부에서 받은 데이터를 함수 내부로 전달하는 역할을 합니다. 이를 통해 함수는 다양한 상황에 맞게 동작할 수 있습니다.
🎈유연성 증가 : 파라미터를 사용함으로써, 하나의 함수로 다양한 입력값에 대해 처리할 수 있으며, 코드의 재사용성과 유지 보수성이 향상되빈다.

{
        function func(str = "17. 실행되었습니다.") {
            console.log(str)
        }
        func();

        const func1 = (str = "18. 실행되었습니다.") => {
            console.log(str)
        }
        func1();
}

이 방식을 통해 객체의 프로토타입에 메서드를 추가하고, 인스턴스를 통해 해당 메서드를 사용할 수 있습니다. * 인스턴스 : 특정 클래스나 함수로부터 생성된 객체 / 클래스나 생성자 함수의 특성을 상속받아, 독립적인 속성 값을 가지는 개별적인 객체가 됩니다.

22. 함수 : 재귀 함수

재귀 함수는 자신을 다시 호출하는 함수로, 자바스크립트에서 복잡한 문제를 간단하게 해결하는 데 유용하게 사용됩니다.
재귀 함수의 핵심은 종료 조건을 설정하여 무한 호출을 방지하는 것입니다. 🔄

✅ 재귀 함수의 기본 개념
정의 : 함수가 자기 자신을 호출하여 문제를 해결하는 방식입니다.
종료 조건 : 재귀 함수는 반드시 종료 조건을 포함해야 하며, 이 조건이 충족될 때 함수 호출이 멈춥니다.

✅ 재귀 함수의 장점과 사용 예시
- 반복적인 문제를 해결할 때 코드를 간결하게 만들 수 있으며, 복잡한 알고리즘을 단순화시키는 데 도움을 줍니다.
- 예시 : pow(x, n)함수를 재귀적으로 구현하여 x의 n 제곱을 계산할 수 있습니다. 이는 반복문을 사용하는 것보다 더 직관적일 수 있습니다.
✅ 재귀 함수의 주의점
- 스택 오버플로우 : 재귀 함수의 호출이 너무 깊어지면, 스택 오버플로우가 발생할 수 있으므로, 종료 조건을 적절히 설정하는 것이 중요합니다. 

{
        // 01 반복문을 사용하는 경우
        function func(num) {
            for (let i = 0; i < num; i++) {
                console.log("19. 실행되었습니다.");
            }
        }
        func(10);

        // 02 재귀함수를 사용하기
        // 동시에 실행시킬때
        function func1(num) {
            if (num < 1) return;

            console.log("20. 실행되었습니다.")
            func1(num - 1); //재귀 호출
        }
        func1(10);
}
결과 확인하기

23. 함수 : 콜백 함수

자바스크립트에서 콜백 함수는 다른 함수에 매개변수로 전달되어, 그 함수의 내부에서 호출되는 함수를 말합니다.
이는 자바스크립트의 비동기 처리 패턴에서 매우 중요한 역할을 합니다.

✅재귀함수와 콜백함수의 차이점
재귀함수는 자기 자신을 호출하며, 주로 반복적인 작업을 간결하게 처리하기 위해 사용되며, 콜백함수는 비동기 처리나 이벤트 리스너 등에 사용됩니다.

✅콜백 함수의 기본 개념
정의 : 다른 함수의 매개변수로 전달되어, 나중에 실행될 함수입니다.

✅콜백 함수의 사용 예시
🎈 이벤트 리스너 : 사용자의 동작에 반응하여 특정 함수를 실행할 때 사용됩니다.
🎈 비동기 처리 : 서버로부터 데이터를 받아온 후 처리할 작업을 정의할 때 사용됩니다.
🎈 고차 함수 : 다른 함수를 매개변수로 받거나 반환하는 함수에서 사용됩니다.

✅콜백 함수의 주의점
콜백 지옥 : 콜백 함수를 과도하게 중첩하여 사용하면 코드의 가독성이 떨어지고, 유지 보수가 어려워질 수 있습니다.
이를 해결하기 위해 프로미스나 async/await 같은 기능을 사용할 수 있습니다. 

{
        // 01. 이벤트

코드는 위에서 아래로 순차적으로 진행되며, 한 작업이 끝나면 다음 작업이 시작됩니다.

🎈 비동기
작업이 순서와 관계없이 여러 작업을 처리할 수 있습니다.
콜백함수, 프로미스, async / await 등을 사용하여 작업이 완료될 때까지 대기하지 않고 다른 작업을 진행할 수 있습니다.

비동기 프로그래밍에서 콜백 함수는 특정 작업이 완료된 후 실행되어야 할 함수를 말합니다.

❗ 비동기 방식에서 콜백 함수를 사용하는 이유
주로 비동기 작업의 완료 후에 수행되어야 할 작업을 정의하기 위해서입니다.
콜백 함수를 사용함으로써, 비동기 작업이 완료되었을 때 특정 동작을 수행하거나 결과를 처리할 수 있습니다.
이를 통해 비동기 작업의 완료 여부를 확인하고, 그에 따른 후속 작업을 수행할 수 있습니다.
콜백함수를 사용하여 비동기 작업의 결과를 효과적으로 처리하고, 코드의 실행 흐름을 관리할 수 있다는 장점을 가지고 있습니다. 


    // 01 동기적인 함수 호출
    {
        function func1() {
            console.log("29. 실행되었습니다.");
        }
        function func2() {
            console.log("30. 실행되었습니다.");
        }
        func2();
        func1();

        // 02 비동기적인 함수 호출
        function func3() {
            setTimeout(() => {
                console.log("31. 실행되었습니다.");
            }, 1000)
        }
        function func4() {
            console.log("32. 실행되었습니다.");
        }
        func3();
        func4();

        // 03 콜백함수 비동기 호출
        // setTimeout(() => {}, 1000) : 1초 후에 함수를 실행시켜라
        {
            function func5(callback) {
                setTimeout(() => {
                    console.log("33. 실행되었습니다.");
                    callback();
                }, 1000);
            }
        }
        function func6() {
            console.log("34. 실행되었습니다.");
        }

        func5(function () {
            func6();
        });
}

32.실행되었습니다.

33.실행되었습니다.
34.실행되었습니다.

26. 함수 : 비동기 방식 : async / await

✅ async / await의 기본 개념

🎈 async함수
함수 앞에 async 키워드를 붙여서 정의합니다. 이 함수는 항상 프로미스(Promise)를 반환합니다.
만약 반환 값이 프로미스가 아니라면, 그 값을 resolve하는 프로미스로 감싸서 반환합니다.

🎈 await 연산자
async함수 내부에서만 사용할 수 있으며, 프로미스의 처리가 끝날 때까지 함수의 실행을 일시 중지합니다.
프로미스가 처리되면 그 결과 값으로 함수 실행을 재개합니다.
async와 await는 자바스크립트에서 비동기 방식을 더 쉽고 직관적으로 다룰 수 있게 해주는 문법입니다.
이를 통해 비동기 코드를 동기 코드처럼 보이게 하여 가독성을 높이고, 코드의 복잡성을 줄일 수 있습니다. 

{
        / 01 : 일반적인 함수(동기방식)
        function func() {
            console.log("35. 실행되었습니다.")
        }
        func();

        / 02 : 함수(비동기방식)
        async function func2() {
            console.log("36. 실행되었습니다.")
        }
        func2();

        / 03 : 비동기 방식(fetch())
        async function func3() {
            const result = await fetch("https://webs9919.github.io/class2024/json/gineungsaJC2005_02.json");
            const data = await result.json();
            console.log(data);
        }
        func3()

        / 04 : 비동기 + 에러
        async function func4() {
            try {
                const result = await fetch("https://webs9919.github.io/class2024/json/gineungsaJC2005_02.json");
                const data = await result.json();
                console.log(data);
            } catch (error) {
                console.error(error);
            }
        }
        func4();
}
결과 확인하기
35. 실행되었습니다. 36. 실행되었습니다.
