12. 배열: 데이터 불러오기: 배열 펼침 연산자(Spread Operator)

배열 구조 분해 할당은 배열의 요소를 개별 변수에 할당하는 JavaScript의 기능입니다.
이를 통해 배열의 각 요소를 쉽게 접근하고 처리할 수 있습니다. 

{
            // 배열 복사
            const arr = [100, 200, "javascript"];
        const copy = [...arr];

        console.log("배열 복사")
        console.log(arr)       // [100, 200, "javascript"];
        console.log(...arr)     // 100 200 "javascript"
        console.log(copy)       // [100, 200, "javascript"];
    }
    {
        // 배열 합치기
        const arr1 = [100, 200];
        const arr2 = [300, 400];
        const merged1 = [...arr1, ...arr2];
        const merged2 = [...arr1, "javascript", ...arr2, "jquery"];

        console.log("배열 합치기")
        console.log(arr1.concat(arr2));         // [100, 200, 300, 400]
        console.log(merged1);       // [100, 200, 300, 400]
        console.log(merged2);       // [100, 200, javascript, 300, 400, jquery]
}
결과 확인하기
배열복사
[100, 200, "javascript"];
100 200 javascript
[100, 200, "javascript"];
배열합치기
[100, 200,300,400];
[300, 200, 300, 400];
[100,200,javascript,300,400,jquery]

18. 객체: 데이터 불러오기: hasOwnProperty()

아래의 예시에서 obj 객체가 "a", "b", "c" 속성을 소유하고 있으므로 각각 true를 반환합니다.
"d" 속성은 obj 객체에 존재하지 않으므로 false를 반환합니다.
in 연산자는 객체가 특정 속성을 소유하고 있는지 확인하는 데 사용됩니다.
아래의 코드에서는 "a", "b", "c" 속성이 obj 객체에 존재하므로 각각 true를 반환합니다.
"d" 속성은 obj 객체에 존재하지 않으므로 false를 반환합니다. 

{
        const obj = {
            a: 100,

        console.log(obj.hasOwnProperty("a"));   // true
        console.log(obj.hasOwnProperty("b"));   // true

        console.log("a" in obj);         // true
        console.log("b" in obj);         // true
        console.log("c" in obj);         // true
        console.log("d" in obj);         // false

21. 객체: 데이터 불러오기: 객체 펼침연산자

객체 펼침 연산자(Spread Operator)는 객체의 속성을 개별적으로 분리하거나, 여러 객체의 속성을 하나의 객체로 합칠 때 사용하는 연산자입니다.
펼침 연산자를 사용하면 두 객체의 속성을 하나의 객체로 합칠 수 있습니다. 

 {
        const obj1 = { a: 100, b: 200 }
        const obj2 = { c: 300, d: 400 }
        const merge1 = { ...obj1, ...obj2 }
        const merge2 = { ...obj1, x: "javascript", ...obj2, y: "jquery" }

        console.log(obj1)
        console.log(merge1);
        console.log(merge2);
}
결과 확인하기
{ a: 100, b: 200 }
{ a: 100, b: 200, c: 300 , d: 400}

XMLHttpRequest(XHR)는 웹 브라우저에서 서버와 데이터를 주고받기 위해 사용되는 객체입니다. 다음과 같은 특징을 가지고 있습니다.

1. 비동기 통신: 서버와 데이터를 주고받을 때 비동기 방식으로 동작합니다.
이는 클라이언트가 서버의 응답을 기다리는 동안에도 다른 작업을 수행할 수 있다는 것을 의미합니다.

2.다양한 데이터 형식 지원: 서버로부터 받은 데이터를 JSON, XML 등 다양한 형식으로 처리할 수 있습니다.

3.상태 확인 가능: 요청이 완료되었는지, 오류가 발생했는지 등의 상태를 확인할 수 있습니다.

4.콜백 함수 사용: 서버로부터 받은 데이터를 처리하기 위해 콜백 함수를 사용합니다.

5.보안 기능 제공: 데이터 전송 시 암호화를 지원하며, 서버의 인증을 요구할 수 있습니다.

{
            {
        var xhr = new XMLHttpRequest();
        xhr.open("GET", "https://webstoryboy.github.io/webs2024/json/gineungsaJC2011_05.json", true);

        xhr.onreadystatechange = function () {
            if (xhr.readyState === 4 && xhr.status === 200) {
                var responseData = xhr.responseText;
                var jsonData = JSON.parse(responseData);

                console.log(jsonData);
            } else if (xhr.readyState === 4) {
                console.error("데이터 불러오기 오류: " + xhr.status);
                        }
                };

        xhr.send();
            }
}

23. 서버 데이터 불러오기: fetch API

fetch API는 자바스크립트에서 서버와 데이터를 주고받기 위한 API입니다.
XMLHttpRequest와 유사하지만, 더 간결하고 사용하기 쉽습니다.
fetch API는 웹 개발에서 매우 유용하게 사용됩니다.
서버와 데이터를 주고받을 때 발생하는 오류를 처리하고, 보안에 신경써야 합니다. 

{
        fetch("https://webstoryboy.github.io/webs2024/json/gineungsaJC2011_05.json")
            .then(response => {
                return response.json();
            })
            .then(data => {
                console.log(data);
            })
            .catch(error => {
                console.error("데이터 불러오기 오류:", error);
            })
}
